# 高度計システム 通信仕様変更書

## 1. 概要
* 本仕様書は、高度計測定システム（LiDARおよび超音波センサ）において、測定データの基本構造（ペイロード）を変更せず、通信識別子（CAN ID）の変更およびI2C通信プロトコルへの対応を行うための設計仕様を定義するものです。

## 2. 共通データ構造（大本プロトコルの維持）
上位システムへのデータ伝送において、メッセージ内のデータ配置（バイト構成）は既存の形式をそのまま維持します。

* **LiDAR（高度メインデータ）**:
  * [cite_start]データ長: 2バイト [cite: 35]
  * [cite_start]Byte 0: 距離データ 上位バイト (`(simulatedLidarRange >> 8) & 0xFF`) [cite: 33]
  * [cite_start]Byte 1: 距離データ 下位バイト (`simulatedLidarRange & 0xFF`) [cite: 34]
* **超音波センサ（近接データ）**:
  * [cite_start]データ長: 2バイト [cite: 36]
  * [cite_start]Byte 0: 距離データ (`simulatedUltSoundDist & 0xFF`) [cite: 36]
  * [cite_start]Byte 1: ステータスフラグ（正常時固定値: `0xAA`） [cite: 37]

## 3. CAN通信仕様の変更
* **通信物理層パラメータ**:
  * [cite_start]通信速度: 500 kbps (`TWAI_TIMING_CONFIG_500KBITS()`) [cite: 18]
  * [cite_start]フレーム形式: 標準ID (11bit) [cite: 1, 25]
* **識別子（CAN ID）の変更**:
  他システムやネットワーク内のデバイスとの競合を回避するため、CAN IDを以下のように変更します。
  * **LiDARデータ**: 
    * [cite_start]変更前: `0x01` [cite: 1] ➔ **変更後: `0x100`** (システム仕様に応じて変更可能)
  * **超音波センサデータ**: 
    * [cite_start]変更前: `0x02` [cite: 2] ➔ **変更後: `0x101`** (システム仕様に応じて変更可能)

## 4. I2C通信プロトコルの定義（通信方式の移行・追加）
CAN通信の代わりに、あるいは並行してI2C通信でデータを取得できるようにするため、大本のデータ構造をレジスタマップ形式に展開して定義します。

* **通信物理層パラメータ**:
  * デバイス配置: ESP32側を **I2Cスレーブ（ペリフェラル）** として設定
  * スレーブアドレス: **`0x30`** (任意に変更可能)
  * 通信速度: Standard Mode (100 kbps) または Fast Mode (400 kbps)
* **I2C レジスタマップ（データ配置仕様）**:
  CANの2バイトずつの構造を、連続したアドレスにマッピングすることでプロトコルの互換性を保ちます。

| レジスタアドレス | データ名 | 割り当て内容 |
| :--- | :--- | :--- |
| `0x00` | LiDAR 距離（上位） | [cite_start]`(simulatedLidarRange >> 8) & 0xFF` [cite: 33] |
| `0x01` | LiDAR 距離（下位） | [cite_start]`simulatedLidarRange & 0xFF` [cite: 34] |
| `0x02` | 超音波 距離 | [cite_start]`simulatedUltSoundDist & 0xFF` [cite: 36] |
| `0x03` | 超音波 ステータス | [cite_start]固定値 `0xAA` [cite: 37] |

* **データ読み出し手順（マスター側からのアクセスプロトコル）**:
  1. I2Cマスターから読み出したい先頭のレジスタアドレス（例: LiDARなら `0x00`）を送信（Write）。
  2. 続けて、マスターから必要なデータ長（例: LiDARデータのみなら2バイト、全データなら4バイト）の送信を要求（`requestFrom`）。
  3. ESP32（スレーブ）は、内部バッファから指定されたレジスタ以降のデータを順次マスターへ返送。
